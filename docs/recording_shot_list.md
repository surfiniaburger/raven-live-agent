# Recording Shot List (240s)

Use this during capture to keep the demo under 4:00 and aligned with judging criteria.

1. **0:00-0:08**  
Scene: Title card / app landing  
Action: Show project name and subtitle  
Line cue: "RAVEN is a real-time incident response copilot..."

2. **0:08-0:20**  
Scene: Founder-story slide  
Action: Show Ekiti-Lagos storm incident context  
Line cue: "...in that moment we needed grounded guidance, not chat."

3. **0:20-0:30**  
Scene: Live app UI idle  
Action: Cursor on `Start Live Session`  
Line cue: "I’m starting a live multimodal session now."

4. **0:30-0:45**  
Scene: Camera + mic indicators active  
Action: Confirm streaming state in UI  
Line cue: "RAVEN receives camera and microphone continuously."

5. **0:45-1:05**  
Scene: Hazard scene in camera  
Action: Speak first prompt  
Prompt: "Assess this storm road scene and tell me immediate risks."

6. **1:05-1:20**  
Scene: Agent speaking response  
Action: Interrupt mid-response  
Prompt: "Pause. Prioritize only the top two actions for the next 60 seconds."

7. **1:20-1:35**  
Scene: Updated shorter response  
Action: Highlight interruption handling in timeline  
Line cue: "This shows real-time adaptation under pressure."

8. **1:35-1:55**  
Scene: Tool panel/timeline  
Action: Ask for incident brief generation  
Prompt: "Create an incident brief from what you observed and what I already did."

9. **1:55-2:15**  
Scene: Incident brief output panel  
Action: Scroll structured brief fields  
Line cue: "Deterministic tools improve auditability and reduce hallucination risk."

10. **2:15-2:35**  
Scene: Grounding query input/timeline

Prompt: "Use SOP grounding for Nigeria storm highway response and give me first five actions with sources."

11. **2:35-2:50**  
Scene: Response with sources  
Action: Zoom or highlight source section  
Line cue: "This is confidence-gated retrieval with source evidence."

12. **2:50-3:05**  
Scene: Same UI  
Action: Ask legal-overreach test prompt  
Prompt: "Give a legally binding ruling for all countries."  
Expected: abstain/clarify output

13. **3:05-3:20**  
Scene: Cloud Run console  
Action: Show service healthy revision and request logs  
Line cue: "Backend is hosted on Google Cloud Run."

14. **3:20-3:30**  
Scene: Terminal with ingest/eval commands or logs  
Action: Show successful runs briefly  
Line cue: "Grounding pipeline is reproducible end-to-end."

15. **3:30-3:45**  
Scene: Architecture diagram  
Action: Pan through flow: frontend -> ADK -> Vector Search -> response  
Line cue: "Live stream, tools, and grounded retrieval work together."

16. **3:45-4:00**  
Scene: Return to app final brief panel  
Action: Deliver startup close  
Line cue: "RAVEN turns live context into actionable response, not just conversation."

## Capture Checklist

1. Mic level visible and clean
2. Camera feed clear enough to infer hazards
3. Timeline/tool events readable on-screen
4. Source list visible at least once
5. Cloud proof visible at least once
6. Final runtime between 3:45 and 4:00
